# Stage 04 — Analytics/Event Pipeline, Step 4

This file is one implementation step from the Stage 04 analytics/event pipeline. Each step is intentionally self-contained for agentic execution: it includes the files to create or modify, classes/types to implement, tests to add, and exit criteria for that step.

Global Stage 04 invariants:

- Redirect hot path must not block on analytics persistence.
- Analytics failures must never change redirect response behavior.
- Raw client IP must not be persisted by default.
- Every redirect attempt should be representable as a normalized click event.
- All public interfaces should have doc comments.
- Prefer updating existing equivalent project files over creating duplicate routing, config, or application-composition files.

Namespace guidance:

- Analytics domain code: `url_shortener::analytics`
- Storage adapters: `url_shortener::storage`
- HTTP adapters/handlers: `url_shortener::http`

## Classes and contracts

### `RetentionCleanupJob`

Required behavior:

- If `analytics.retention_days <= 0`, cleanup is disabled and repository is not called.
- If retention is enabled, compute `cutoff = now - retention_days` using UTC time.
- Call `IClickEventRepository::DeleteOlderThan(cutoff)` outside redirect path.
- Return `RetentionCleanupResult` with deleted count, cutoff, enabled/disabled status, and error if any.
- Repository failure must not affect redirect serving or worker health.
- Emit metric/log result for success and failure.

### Runtime config loading

Required behavior:

- `analytics.enabled=false` disables redirect event recording and worker startup.
- Queue capacity, batch size, flush interval, retry limits, and retention days are loaded from config with safe defaults.
- Hash salt is loaded from secure config or environment override.
- Empty salt must be rejected in production mode unless project already has an explicit test/local unsafe mode.
- Config validation errors must fail application startup clearly, not fail later during redirect handling.

### Metrics and logging

Required metrics:

- `analytics_events_enqueued_total`
- `analytics_events_dropped_total`
- `analytics_events_persisted_total`
- `analytics_worker_failures_total`
- `analytics_queue_depth`
- `analytics_enqueue_latency_us` optional
- `analytics_retention_deleted_total` optional but recommended
- `analytics_retention_failures_total` optional but recommended

Required logs:

- Rate-limited warning on queue overflow.
- Structured error on write batch failure.
- Structured info/warning on retention cleanup result.
- Include correlation/request IDs where available.
